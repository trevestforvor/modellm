# Phase 2 вҖ” UI Design Contract: HF Browse + Compatibility UI

> Visual and interaction contract for Phase 2. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Platform: iOS (SwiftUI, iOS 17+). Not a web project вҖ” shadcn is not applicable.

---

## Design System

| Property | Value | Source |
|----------|-------|--------|
| Tool | none (native SwiftUI) | Platform constraint вҖ” iOS SwiftUI app |
| Preset | not applicable | вҖ” |
| Component library | SwiftUI native + SF Symbols | CLAUDE.md: SwiftUI iOS 17+ target |
| Icon library | SF Symbols 5 | Apple native, zero dependency |
| Font | SF Pro (system default via `.font(.body)` etc.) | iOS system font вҖ” never declare a custom font |
| Color system | Semantic system colors + explicit semantic tokens below | Ensures automatic dark mode support |

> Do NOT use hex literals for background/text/secondary colors. Use `Color(.systemBackground)`, `Color(.secondarySystemBackground)`, `Color(.label)`, `Color(.secondaryLabel)` etc. Only the compatibility tier colors below are explicit hex values.

## Spacing Scale

All values in SwiftUI points (1pt = 1px on non-retina; use these as SwiftUI spacing constants).

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon-to-label gap, pill internal horizontal padding |
| sm | 8pt | Card internal element vertical spacing, badge padding |
| md | 16pt | Default HStack/VStack spacing, card horizontal padding |
| lg | 24pt | Section header bottom margin, card vertical padding |
| xl | 32pt | Scroll view top padding, inter-section spacing |
| 2xl | 48pt | Empty state illustration-to-text gap |
| 3xl | 64pt | Not used in this phase |

Exceptions:
- Touch targets: minimum 44pt height on all tappable elements (Apple HIG requirement). Applies to variant rows in detail view.
- Recommendation card width: full width minus 2Г—md (16pt) horizontal inset on each side.
- Bottom safe area: always use `.safeAreaInset(edge: .bottom)` вҖ” never hardcode.

---

## Typography

All roles map to SwiftUI `Font` API вҖ” never use raw point sizes directly in views. Use the named role constants below.

| Role | SwiftUI Font | Approx Size | Weight | Line Height | Usage |
|------|-------------|-------------|--------|-------------|-------|
| Display | `.title2` | 22pt | `.semibold` (600) | 1.2 | Section headers ("Recommended for your device", "All Models") |
| Heading | `.headline` | 17pt | `.semibold` (600) | 1.2 | Model name on card and detail header |
| Body | `.body` | 17pt | `.regular` (400) | 1.5 | Model description in detail view |
| Label | `.subheadline` | 15pt | `.regular` (400) | 1.4 | Card metadata row (size, params, quant type, download count) |
| Caption | `.caption` | 12pt | `.regular` (400) | 1.3 | Secondary detail labels ("Uses X GB, you have Y GB free"), pill badge text |

Weight contract: exactly 2 weights вҖ” `.regular` (400) and `.semibold` (600). No `.bold`, `.medium`, or `.light`.

---

## Color

Compatibility tier colors are the ONLY explicit semantic colors in this phase. All other colors use system semantics.

| Role | Value | Usage |
|------|-------|-------|
| Dominant surface (60%) | `Color(.systemBackground)` | App background, navigation bar |
| Secondary surface (30%) | `Color(.secondarySystemBackground)` | Model cards, detail sheet background |
| Tertiary surface | `Color(.tertiarySystemBackground)` | Variant rows in detail list |
| Accent вҖ” Runs Well | `#34C759` (system green) or `Color.green` | tok/s badge background for .runsWell tier ONLY |
| Accent вҖ” Runs Slowly | `#FF9F0A` (system orange) or `Color.orange` | tok/s badge background for .runsSlow tier ONLY |
| Destructive | `Color.red` / `Color(.systemRed)` | Not used in Phase 2 (no destructive actions) |
| Text primary | `Color(.label)` | All primary text |
| Text secondary | `Color(.secondaryLabel)` | Metadata labels, captions |

Accent reserved for:
- tok/s pill badge background (green = .runsWell, orange = .runsSlow) вҖ” these are the ONLY colored accents in the browse UI
- Badge text is always `.white` at `.caption` weight

Do NOT use accent color for buttons, links, or any other interactive chrome. Use `Color(.label)` for navigation actions and `Color(.systemBlue)` only if a hyperlink to HF is required in the detail view.

## Component Contracts

### Tok/s Compatibility Badge (pill)

The centerpiece interaction element вҖ” appears on every model card and every variant row.

```
[ ~25 tok/s ]   вҶҗ green background (#34C759), white caption text, 6pt corner radius
[ ~8 tok/s  ]   вҶҗ orange background (#FF9F0A), white caption text, 6pt corner radius
```

- Width: intrinsic (hugs content)
- Padding: 4pt vertical, 8pt horizontal (xs vertical, sm horizontal)
- Corner radius: 6pt
- Font: `.caption` `.semibold` вҖ” exception to the 2-weight rule: badge uses semibold caption for legibility at small size
- Source: `CompatibilityResult.estimatedToksPerSec` formatted as "~\(N) tok/s"
- Never show a badge for `.incompatible` tier вҖ” those models are hidden (D-05 from CONTEXT.md)

### Model Card

Card layout (vertical stack inside a rounded rectangle):

```
в”Ңв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”җ
в”Ӯ  Model Name (Heading, semibold)         в”Ӯ
в”Ӯ  7B В· Q4_K_M В· 4.1 GB В· 12.4K downloadsв”Ӯ  вҶҗ Label, secondary color, single line
в”Ӯ                              [ ~25 tok/s]в”Ӯ  вҶҗ Badge, trailing aligned
в””в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
```

- Background: `Color(.secondarySystemBackground)`
- Corner radius: 12pt
- Padding inside card: md (16pt) horizontal, sm+md (12pt) vertical
- Card spacing in list: sm (8pt) between cards
- Full-width layout: `.listStyle(.plain)` or `LazyVStack` with horizontal padding md (16pt) each side
- Metadata row: HStack with `.subheadline` and `Color(.secondaryLabel)`, separated by `В·` character (not bullet)
- Badge: trailing within card, vertically centered with model name

```

[Card] [Card] [Card] [Card]       вҶҗ Horizontal scroll, 4вҖ“6 cards

"All Models"                      вҶҗ Display title
[Search bar]
[Card]
[Card]

```

- Recommendation cards scroll horizontally вҖ” use `ScrollView(.horizontal, showsIndicators: false)` with a `LazyHStack`
- Recommendation card width: screen width Г— 0.8, max 320pt вҖ” shows edge of next card to signal scrollability
- "All Models" section follows immediately below in the same ScrollView (vertical outer scroll)
- Search bar: `searchable(text:)` modifier on NavigationStack вҖ” not a custom view

### Detail View

Navigation pattern: **NavigationStack push** (not sheet). Rationale: detail is a full-screen destination with a back button; a sheet implies dismissal is primary, but on detail the user reads and then decides. Push is the right affordance for browse вҶ’ detail вҶ’ (future) download flow.

Detail screen layout:

```

  [Model card text from HF, body font, 3 lines max with "Read more"]

в”Җв”Җ Storage Impact в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
  Uses 4.1 GB В· You have 6.1 GB free

в”Җв”Җ Variants в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ

  [Download вҖ” Coming in Phase 3]  вҶҗ disabled button, full width
```

- Section headers: `.headline` weight `.semibold`, `Color(.secondaryLabel)` вҖ” uppercase NOT used
- Variant rows: `List` rows with `.listStyle(.insetGrouped)` per section
- Variant row height: minimum 44pt (touch target compliance)
- Storage impact: `Color(.secondaryLabel)` caption below description section
- Download button: full-width, `.buttonStyle(.borderedProminent)`, disabled state, label "Download В· Coming Soon"
- Tapping a variant row does nothing in Phase 2 (download is Phase 3)

## Interaction States

### Loading State

Use skeleton shimmer for card list while HF API responds. Implementation: overlay `Color(.secondarySystemBackground)` rounded rectangles matching card dimensions, animated with `.opacity` between 0.4 and 1.0 using `withAnimation(.easeInOut(duration: 0.8).repeatForever(autoreverses: true))`. No third-party shimmer library.

- Show skeleton immediately on view appear
- Skeleton shows 3 placeholder cards at correct dimensions
- Replace with real cards when first page loads

### Error State

Network failure or HF API error:

```
[SF Symbol: wifi.exclamationmark, .largeTitle scale]
"Couldn't load models"
"Check your connection and try again."
[Try Again]  вҶҗ .bordered button style
```

- Symbol: `wifi.exclamationmark` or `exclamationmark.triangle`
- Body text: `.body` regular, `Color(.secondaryLabel)`, centered
- Button: `"Try Again"`, `.bordered` button style (not prominent вҖ” not a happy path)

### Empty State (no search results)

```
[SF Symbol: magnifyingglass, .largeTitle scale]
"No models found"
"Try a different search term or browse all models."
[Browse All Models]  вҶҗ .bordered button, clears search
```

### Pagination / Infinite Scroll

Append next page when user scrolls within 3 cards of list end. Show a `ProgressView()` at list bottom during page load. No explicit "Load More" button.

  в””в”Җв”Җ HomeView (default tab or root)
        в”ңв”Җв”Җ Recommendations (LazyHStack horizontal scroll)
        в””в”Җв”Җ All Models (LazyVStack, searchable)

## Accessibility

- All compatibility badges must have `.accessibilityLabel("Runs at approximately \(N) tokens per second")`
- Model cards must have `.accessibilityElement(children: .combine)` so VoiceOver reads the card as a unit
- Disabled download button: `.accessibilityHint("Download available in a future update")`
- Minimum touch target 44pt enforced via `.frame(minHeight: 44)` on variant rows
- Dynamic Type: all font roles use system fonts вҖ” Dynamic Type support is automatic
