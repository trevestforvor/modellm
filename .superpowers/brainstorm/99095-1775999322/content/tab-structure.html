Tab Structure: How Many Tabs?

Every competitor uses 2-3 primary destinations max. Four tabs feels bloated for this app's scope.

Current ModelRunner has 4 tabs: Browse (HF search) + Library (downloaded) + Chat + Settings (just server management). The research shows competitors consolidate aggressively.

A

Two Tabs: Models + Chat

Models — one unified view with sections: "My Models" (downloaded + remote servers) at top, "Browse Hugging Face" below. Server management via "+" button. Grid of cards for My Models, list for Browse results.

Chat — conversation view with model picker.

This is what PocketPal and Enchanted do. Simplest possible navigation. Settings goes into a gear icon or long-press on models.

B

Three Tabs: Models + Chat + Settings

Models — same unified view as Option A (local + remote + browse in sections).

Chat — conversation view.

Settings — app preferences, inference defaults, about screen. NOT server management (that lives in Models).

Keeps a dedicated place for future settings (theme, default params, notifications). More room to grow.

C

Three Tabs: Discover + Library + Chat

Discover — browse HF models with compatibility badges (the core differentiator).

Library — "My stuff": downloaded models as cards + remote servers as cards. Unified "what I can use right now" view. Server "+" button here.

Chat — conversation view.

LM Studio pattern. Separates "finding new models" from "using what I have." Settings via gear icon in Library or Chat.

My Recommendation: Option A

Two tabs is bold but right. The current "Browse" tab is barely functional (HF API not wired), and "Library" is empty (no downloads yet). Merging them into one "Models" tab with sections gives you a single, powerful home screen. Settings can live behind a gear icon — you don't need a whole tab for server URLs and inference defaults.
